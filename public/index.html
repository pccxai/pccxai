What this site is

A quiet front door for the ecosystem

The organization is split across focused repositories instead of a single monolith. This page points to the current public surfaces: the pccx documentation, the active FPGA repository, lab tooling, a local launcher scaffold, and SystemVerilog editor work.

Hardware-sensitive statements are kept conservative here. Build, timing, bitstream, throughput, and board-level inference claims belong with reproducible evidence in the repository that produced them.

pccx at a high level

A hardware/software co-design track

PCCX stands for Parallel Compute Core eXecutor. At a high level, pccx explores an NPU architecture and supporting software stack for transformer decoding workloads on edge FPGA-class systems.

The public work covers architecture documentation, RTL sources, trace and verification workflows, and tooling that can consume those interfaces. The project is still evidence-first: prototypes are described as prototypes, and release language follows the verification status of each repository.
